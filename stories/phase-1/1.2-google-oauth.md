# Story 1.2 — Google OAuth via NextAuth.js

**Phase:** 1 — Foundation & Auth
**Depends on:** Story 1.1 (project scaffolded)
**Estimated time:** 1–2 hours

---

## Context

ARIA needs Google OAuth to access Gmail and Google Calendar on Doug's behalf. NextAuth.js v5 handles the OAuth flow, token storage, and refresh automatically.

After this story, visiting ARIA will require signing in with Google, and the session will include the access token needed to call Gmail and Calendar APIs.

---

## What to Build

- Install and configure NextAuth.js v5 with the Google provider
- Configure the correct OAuth scopes for Gmail (read + send) and Google Calendar (read + write)
- Protect the dashboard route — unauthenticated users are redirected to sign-in
- Display the signed-in user's name and avatar in a top-right corner of the dashboard
- Provide a sign-out button

---

## Technical Instructions

### Step 1: Install dependencies

```bash
npm install next-auth@beta
```

### Step 2: Create auth config

Create `lib/auth.ts`:

```typescript
import NextAuth from 'next-auth';
import Google from 'next-auth/providers/google';

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [
    Google({
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      authorization: {
        params: {
          scope: [
            'openid',
            'email',
            'profile',
            'https://www.googleapis.com/auth/gmail.readonly',
            'https://www.googleapis.com/auth/gmail.send',
            'https://www.googleapis.com/auth/gmail.modify',
            'https://www.googleapis.com/auth/calendar',
          ].join(' '),
          access_type: 'offline',   // get refresh token
          prompt: 'consent',        // force consent screen to always get refresh token
        },
      },
    }),
  ],
  callbacks: {
    async jwt({ token, account }) {
      // Store access + refresh tokens in the JWT on first sign-in
      if (account) {
        token.accessToken = account.access_token;
        token.refreshToken = account.refresh_token;
        token.expiresAt = account.expires_at;
      }
      return token;
    },
    async session({ session, token }) {
      // Expose access token to server-side API routes via session
      session.accessToken = token.accessToken as string;
      return session;
    },
  },
});
```

### Step 3: Create the NextAuth API route

Create `app/api/auth/[...nextauth]/route.ts`:

```typescript
import { handlers } from '@/lib/auth';
export const { GET, POST } = handlers;
```

### Step 4: Extend NextAuth types

Create `types/next-auth.d.ts`:

```typescript
import { DefaultSession } from 'next-auth';

declare module 'next-auth' {
  interface Session {
    accessToken?: string;
  }
}
```

### Step 5: Create middleware to protect the dashboard

Create `middleware.ts` in the project root:

```typescript
export { auth as middleware } from '@/lib/auth';

export const config = {
  matcher: ['/((?!api/auth|_next/static|_next/image|favicon.ico).*)'],
};
```

### Step 6: Update dashboard to show user info

Update `app/page.tsx` to display session info and a sign-out button. Import `auth` from `@/lib/auth` and use it as a server component:

```tsx
import { auth, signOut } from '@/lib/auth';
import { redirect } from 'next/navigation';

export default async function DashboardPage() {
  const session = await auth();
  if (!session) redirect('/api/auth/signin');

  return (
    <main className="min-h-screen bg-background text-foreground p-8">
      <div className="flex justify-between items-center mb-8">
        <div>
          <h1 className="text-3xl font-bold">ARIA</h1>
          <p className="text-muted-foreground">AI-Powered Personal Assistant</p>
        </div>
        <div className="flex items-center gap-4">
          <span className="text-sm text-muted-foreground">
            {session.user?.name}
          </span>
          <form action={async () => {
            'use server';
            await signOut();
          }}>
            <button type="submit" className="text-sm text-muted-foreground hover:text-foreground">
              Sign out
            </button>
          </form>
        </div>
      </div>
      <p className="text-muted-foreground">Dashboard loading... (Phase 1)</p>
    </main>
  );
}
```

### Step 7: Add environment variables to .env.local

Ensure `.env.local` contains:
```
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<generated with openssl rand -base64 32>
GOOGLE_CLIENT_ID=<from Google Cloud Console>
GOOGLE_CLIENT_SECRET=<from Google Cloud Console>
```

See `docs/api-setup.md` for how to get Google credentials.

## Notes for Claude Code

- NextAuth v5 (beta) has a different API than v4 — use `auth()` server-side, not `getServerSession()`
- The `access_type: 'offline'` + `prompt: 'consent'` combo is required to get a refresh token from Google
- If you get a "redirect_uri_mismatch" error, the Google Cloud Console redirect URI must exactly match `http://localhost:3000/api/auth/callback/google`
- Never log or expose the full access token in the UI
