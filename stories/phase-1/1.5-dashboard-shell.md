# Story 1.5 — Dashboard Shell & Widget Grid

**Phase:** 1 — Foundation & Auth
**Depends on:** Stories 1.1–1.4
**Estimated time:** 2–3 hours

---

## Context

ARIA's UI is a widget-based dashboard — not a page-based app. All five modules (Email, Calendar, Tasks, Notes, Morning Briefing) are panels on a single screen, similar to a mission control dashboard. The shell is the structural layout that houses all widgets.

This story builds the visual shell: the top nav, sidebar, and widget grid. All widgets are empty placeholders for now — real data comes in Phase 2 and 3.

The design target is **minimalist dark mode** with clean typography. Think Linear or Raycast — not a colorful BI dashboard.

---

## Technical Instructions

### Layout Design

```
┌────────────────────────────────────────────────────┐
│ TopBar: [ARIA logo]                [user] [settings]│
├──────┬─────────────────────────────────────────────┤
│      │ Morning Briefing Widget (full width)         │
│      ├───────────────────────┬─────────────────────┤
│ Side │ Email Widget          │ Calendar Widget      │
│ bar  │                       │                      │
│      ├───────────────────────┼─────────────────────┤
│      │ Task Widget           │ Notes Widget         │
│      │                       │                      │
└──────┴───────────────────────┴─────────────────────┘
```

### Step 1: Create DashboardShell

Create `components/DashboardShell.tsx`:

```tsx
import { TopBar } from './TopBar';
import { Sidebar } from './Sidebar';

interface DashboardShellProps {
  children: React.ReactNode;
  userName?: string;
  userEmail?: string;
}

export function DashboardShell({ children, userName, userEmail }: DashboardShellProps) {
  return (
    <div className="flex h-screen bg-background text-foreground overflow-hidden">
      <Sidebar />
      <div className="flex-1 flex flex-col overflow-hidden">
        <TopBar userName={userName} userEmail={userEmail} />
        <main className="flex-1 overflow-auto p-4">
          {children}
        </main>
      </div>
    </div>
  );
}
```

### Step 2: Create TopBar

Create `components/TopBar.tsx`. Include:
- ARIA logo/wordmark on the left
- Current date (e.g., "Monday, April 13") in the center
- User initials/avatar on the right with a dropdown containing Settings and Sign Out

### Step 3: Create Sidebar

Create `components/Sidebar.tsx`. Include icon buttons for:
- Dashboard (home icon)
- Email (mail icon)
- Calendar (calendar icon)
- Tasks (checkmark/list icon)
- Notes (file-text icon)

Use Lucide icons (already available via shadcn/ui). The sidebar should be narrow (icon-only, ~56px wide) with tooltips on hover showing the label.

### Step 4: Create placeholder widgets

Each widget should be a shadcn `Card` with:
- A header showing the module name and a subtle icon
- A placeholder content area with a loading skeleton or "Not yet connected" message
- A consistent height

Create these files with the placeholder pattern:

`components/widgets/MorningBriefingWidget.tsx`:
```tsx
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card';
import { Sun } from 'lucide-react';

export function MorningBriefingWidget() {
  return (
    <Card className="col-span-2 mb-4">
      <CardHeader className="pb-2">
        <CardTitle className="text-sm font-medium flex items-center gap-2 text-muted-foreground">
          <Sun className="h-4 w-4" />
          Morning Briefing
        </CardTitle>
      </CardHeader>
      <CardContent>
        <p className="text-sm text-muted-foreground">
          Your AI-generated briefing will appear here on load...
        </p>
      </CardContent>
    </Card>
  );
}
```

Follow the same pattern for `EmailWidget`, `CalendarWidget`, `TaskWidget`, `NotesWidget` — adjust the icon and label for each.

### Step 5: Assemble in app/page.tsx

```tsx
import { auth } from '@/lib/auth';
import { redirect } from 'next/navigation';
import { DashboardShell } from '@/components/DashboardShell';
import { MorningBriefingWidget } from '@/components/widgets/MorningBriefingWidget';
import { EmailWidget } from '@/components/widgets/EmailWidget';
import { CalendarWidget } from '@/components/widgets/CalendarWidget';
import { TaskWidget } from '@/components/widgets/TaskWidget';
import { NotesWidget } from '@/components/widgets/NotesWidget';

export default async function DashboardPage() {
  const session = await auth();
  if (!session) redirect('/api/auth/signin');

  return (
    <DashboardShell
      userName={session.user?.name ?? undefined}
      userEmail={session.user?.email ?? undefined}
    >
      <MorningBriefingWidget />
      <div className="grid grid-cols-2 gap-4">
        <EmailWidget />
        <CalendarWidget />
        <TaskWidget />
        <NotesWidget />
      </div>
    </DashboardShell>
  );
}
```

## Design Notes for Claude Code

- Use shadcn/ui `Card`, `CardHeader`, `CardTitle`, `CardContent` for all widgets
- Sidebar width: `w-14` (56px), fixed, not collapsible in v1
- All text should use `text-muted-foreground` for secondary content, `text-foreground` for primary
- Use `gap-4` for widget spacing, `p-4` for main content padding
- Do NOT add real data or API calls — this is layout only
- Don't add animations yet — that's Phase 4
