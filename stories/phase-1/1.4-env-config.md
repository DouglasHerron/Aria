# Story 1.4 — Environment Configuration & Validation

**Phase:** 1 — Foundation & Auth
**Depends on:** Stories 1.1, 1.2, 1.3
**Estimated time:** 30 minutes

---

## Context

ARIA depends on many environment variables (API keys, vault paths, etc.). Missing or misconfigured variables cause cryptic runtime errors. This story adds a startup validation layer that checks all required env vars on server boot and fails fast with a clear error message.

---

## What to Build

- A `lib/config.ts` module that reads, validates, and exports all environment variables as a typed config object
- A startup check that throws a clear error listing exactly which vars are missing
- A `/api/health` route that reports config status (without exposing values)

---

## Technical Instructions

### Step 1: Create lib/config.ts

```typescript
// lib/config.ts
// Single source of truth for all environment configuration.
// Import from here, never from process.env directly.

function requireEnv(key: string): string {
  const value = process.env[key];
  if (!value) {
    throw new Error(`[ARIA Config] Missing required environment variable: ${key}\nCheck .env.local — see .env.example for reference.`);
  }
  return value;
}

function optionalEnv(key: string, defaultValue: string): string {
  return process.env[key] ?? defaultValue;
}

export const config = {
  // Auth
  nextAuthUrl: requireEnv('NEXTAUTH_URL'),
  nextAuthSecret: requireEnv('NEXTAUTH_SECRET'),

  // Google
  googleClientId: requireEnv('GOOGLE_CLIENT_ID'),
  googleClientSecret: requireEnv('GOOGLE_CLIENT_SECRET'),

  // AI
  aiProvider: optionalEnv('AI_PROVIDER', 'claude') as 'claude' | 'openai',
  anthropicApiKey: optionalEnv('ANTHROPIC_API_KEY', ''),
  anthropicModel: optionalEnv('ANTHROPIC_MODEL', 'claude-opus-4-6'),
  openaiApiKey: optionalEnv('OPENAI_API_KEY', ''),
  openaiModel: optionalEnv('OPENAI_MODEL', 'gpt-4o'),

  // Obsidian
  obsidianVaultPath: optionalEnv('OBSIDIAN_VAULT_PATH', '/Users/doug/Obsidian'),
  obsidianRestApiPort: parseInt(optionalEnv('OBSIDIAN_REST_API_PORT', '27123')),
  obsidianRestApiKey: optionalEnv('OBSIDIAN_REST_API_KEY', ''),

  // Apple Calendar
  appleCaldavUsername: optionalEnv('APPLE_CALDAV_USERNAME', ''),
  appleCaldavPassword: optionalEnv('APPLE_CALDAV_PASSWORD', ''),
  appleCaldavUrl: optionalEnv('APPLE_CALDAV_URL', 'https://caldav.icloud.com'),

  // Feature flags
  demoMode: optionalEnv('DEMO_MODE', 'false') === 'true',
} as const;

// Validate that the active AI provider has its key set
export function validateAIConfig() {
  if (config.aiProvider === 'claude' && !config.anthropicApiKey) {
    throw new Error('[ARIA Config] AI_PROVIDER=claude but ANTHROPIC_API_KEY is not set');
  }
  if (config.aiProvider === 'openai' && !config.openaiApiKey) {
    throw new Error('[ARIA Config] AI_PROVIDER=openai but OPENAI_API_KEY is not set');
  }
}
```

### Step 2: Create /api/health route

Create `app/api/health/route.ts`:

```typescript
import { NextResponse } from 'next/server';
import { config } from '@/lib/config';

export async function GET() {
  // Return config status without exposing actual values
  return NextResponse.json({
    status: 'ok',
    timestamp: new Date().toISOString(),
    config: {
      aiProvider: config.aiProvider,
      aiModel: config.aiProvider === 'claude' ? config.anthropicModel : config.openaiModel,
      hasAnthropicKey: !!config.anthropicApiKey,
      hasOpenAIKey: !!config.openaiApiKey,
      obsidianVaultPath: config.obsidianVaultPath,
      hasObsidianKey: !!config.obsidianRestApiKey,
      hasAppleCalDAV: !!config.appleCaldavUsername,
      demoMode: config.demoMode,
    },
  });
}
```

### Step 3: Update AI providers to use config

Update `lib/ai/providers/claude.ts` and `lib/ai/providers/openai.ts` to import from `@/lib/config` instead of using `process.env` directly.

In `claude.ts`:
```typescript
import { config } from '@/lib/config';
const client = new Anthropic({ apiKey: config.anthropicApiKey });
const MODEL = config.anthropicModel;
```

In `openai.ts`:
```typescript
import { config } from '@/lib/config';
const client = new OpenAI({ apiKey: config.openaiApiKey });
const MODEL = config.openaiModel;
```

## Notes for Claude Code

- `requireEnv()` only runs server-side (in API routes), so it's safe to throw errors
- Don't expose actual key values in the health endpoint — only boolean "has key" status
- `config` is `as const` so TypeScript narrows the types correctly
