# Story 1.1 — Next.js + Tailwind + shadcn/ui Scaffold

**Phase:** 1 — Foundation & Auth
**Depends on:** Nothing — this is the first story
**Estimated time:** 1–2 hours

---

## Context

You are building ARIA — an AI-powered personal assistant dashboard for Doug. ARIA consolidates email, calendar, tasks, and notes into one interface. The AI runs automatically in the background; there is no chat UI.

This story scaffolds the entire project from scratch using Next.js 14 (App Router), Tailwind CSS, and shadcn/ui.

**Read `CLAUDE.md` in the project root before starting.** It contains the full project overview and target folder structure.

---

## What to Build

Scaffold and configure the base Next.js project so that:
- The project runs at `http://localhost:3000`
- Tailwind CSS is working (test with a colored div)
- shadcn/ui is initialized with at least 3 components installed
- The root `app/page.tsx` renders a placeholder dashboard shell (just a heading and "ARIA is loading..." text for now)
- TypeScript is in strict mode
- The `lib/`, `components/widgets/`, `types/`, and `hooks/` directories exist with placeholder index files

## Technical Instructions

### Step 1: Scaffold Next.js

```bash
npx create-next-app@latest aria-dashboard \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir=no \
  --import-alias="@/*"
cd aria-dashboard
```

### Step 2: Initialize shadcn/ui

```bash
npx shadcn@latest init
```

When prompted:
- Style: **Default**
- Base color: **Slate**
- CSS variables: **Yes**

Then add the components ARIA will use:

```bash
npx shadcn@latest add button card badge separator scroll-area dialog command
```

### Step 3: Create the project directory structure

Create the following directories and placeholder files:

```
lib/
lib/ai/
lib/ai/providers/
lib/ai/prompts/
components/widgets/
hooks/
types/
```

Create `types/index.ts` with this starter content:

```typescript
// ARIA — Shared TypeScript types
// Add module-specific types here as stories are implemented

export type AIProvider = 'claude' | 'openai';

export interface EmailThread {
  id: string;
  subject: string;
  from: string;
  snippet: string;
  date: string;
  unread: boolean;
  // AI-enriched fields (populated after AI processing)
  summary?: string;
  urgency?: 'high' | 'medium' | 'low';
  category?: 'action_needed' | 'fyi' | 'archive';
  draftReply?: string;
}

export interface CalendarEvent {
  id: string;
  title: string;
  start: string;
  end: string;
  source: 'google' | 'apple';
  location?: string;
  description?: string;
  // AI-enriched
  agendaSummary?: string;
  hasConflict?: boolean;
}

export interface ObsidianTask {
  id: string;
  text: string;
  completed: boolean;
  filePath: string;
  lineNumber: number;
  dueDate?: string;
  tags?: string[];
  // AI-enriched
  priorityScore?: number;
  focusRank?: number; // 1, 2, or 3 if in today's top 3
}

export interface ObsidianNote {
  path: string;
  title: string;
  content: string;
  folder: string;
  created: string;
  modified: string;
  tags?: string[];
}

export interface MorningBriefing {
  generatedAt: string;
  summary: string; // Full AI-generated briefing text
  topTasks: ObsidianTask[];
  todayEvents: CalendarEvent[];
  urgentEmails: EmailThread[];
}
```

### Step 5: Update app/globals.css

Ensure dark mode is the default by adding to `app/layout.tsx`:

```tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className="dark">
      <body>{children}</body>
    </html>
  );
}
```
