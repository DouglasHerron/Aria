## Context

Right now the Email Widget shows raw Gmail threads — subjects and snippets. This story wires in the AI layer: for each thread, the AI reads the full message body and returns a structured JSON object with a summary, urgency score, category classification, and a pre-drafted reply.

This is the heart of ARIA's email intelligence — not a chatbot, but a structured AI processing pipeline that runs automatically on load.

## Technical Instructions

### Step 1: Add full body fetch to lib/gmail.ts

```typescript
export async function fetchThreadBody(
  accessToken: string,
  threadId: string
): Promise<string> {
  const gmail = getGmailClient(accessToken);
  
  const response = await gmail.users.threads.get({
    userId: 'me',
    id: threadId,
    format: 'full',
  });

  const messages = response.data.messages ?? [];
  const latestMessage = messages[messages.length - 1];
  
  // Extract text body from MIME parts
  return extractTextFromPayload(latestMessage?.payload);
}

function extractTextFromPayload(payload: any): string {
  if (!payload) return '';
  
  // Direct text/plain part
  if (payload.mimeType === 'text/plain' && payload.body?.data) {
    return Buffer.from(payload.body.data, 'base64').toString('utf-8');
  }
  
  // Multipart — recurse into parts
  if (payload.parts) {
    for (const part of payload.parts) {
      const text = extractTextFromPayload(part);
      if (text) return text;
    }
  }
  
  return '';
}
```

### Step 2: Create lib/ai/prompts/email.ts

```typescript
import { ai } from '@/lib/ai';
import type { EmailThread } from '@/types';

const EMAIL_SYSTEM_PROMPT = `You are ARIA's email intelligence processor. Your job is to analyze email threads and return structured JSON.

You must ALWAYS return valid JSON with this exact structure:
{
  "summary": "1-2 sentence summary of what this email is about and what action (if any) is needed",
  "urgency": "high" | "medium" | "low",
  "category": "action_needed" | "fyi" | "archive",
  "draftReply": "A complete, professional reply that Doug can send with one click. Match the tone of the original email."
}

Urgency guidelines:
- high: Requires action within 24 hours, or sender is waiting for a response
- medium: Requires action this week
- low: FYI, no action needed, or can be archived

Category guidelines:  
- action_needed: Doug needs to respond or do something
- fyi: Informational, no action required
- archive: Promotions, notifications, newsletters, auto-generated emails`;

export async function processEmailThread(
  thread: Pick<EmailThread, 'subject' | 'from' | 'snippet'>,
  body: string
): Promise<{
  summary: string;
  urgency: 'high' | 'medium' | 'low';
  category: 'action_needed' | 'fyi' | 'archive';
  draftReply: string;
}> {
  const prompt = `Analyze this email thread and return a JSON response:

Subject: ${thread.subject}
From: ${thread.from}
Body:
${body.slice(0, 2000)}`; // Truncate to 2000 chars to control costs

  const result = await ai.complete({
    system: EMAIL_SYSTEM_PROMPT,
    prompt,
    maxTokens: 512,
  });

  try {
    // Strip markdown code fences if present
    const json = result.text.replace(/```json\n?|\n?```/g, '').trim();
    return JSON.parse(json);
  } catch {
    // Fallback if JSON parsing fails
    return {
      summary: thread.snippet,
      urgency: 'medium',
      category: 'action_needed',
      draftReply: `Hi,\n\nThank you for your email. I'll get back to you shortly.\n\nBest,\nDoug`,
    };
  }
}
```

### Step 3: Update the threads API route

In `app/api/email/threads/route.ts`, update the GET handler to:
1. Fetch raw threads as before
2. For each thread, fetch the body AND run AI enrichment in parallel (limit to top 10 threads to control API costs)
3. Return enriched threads

```typescript
import { fetchInboxThreads, fetchThreadBody } from '@/lib/gmail';
import { processEmailThread } from '@/lib/ai/prompts/email';

// In GET handler, after fetching threads:
const rawThreads = await fetchInboxThreads(session.accessToken, 10);

// Enrich with AI in parallel
const enrichedThreads = await Promise.all(
  rawThreads.map(async thread => {
    try {
      const body = await fetchThreadBody(session.accessToken, thread.id);
      const aiData = await processEmailThread(thread, body);
      return { ...thread, ...aiData };
    } catch (error) {
      console.error(`AI processing failed for thread ${thread.id}:`, error);
      return thread; // Return un-enriched if AI fails
    }
  })
);

return NextResponse.json({ threads: enrichedThreads });
```
