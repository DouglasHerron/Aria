# Story 3.4 — Morning Briefing Generator

**Phase:** 3 — AI Intelligence Layer
**Depends on:** Stories 3.1, 3.2, 3.3 (all AI processors), Story 1.5 (dashboard shell)
**Estimated time:** 1–2 hours

---

## Context

The Morning Briefing is ARIA's showcase feature — a full-width widget at the top of the dashboard that auto-generates a personalized daily briefing by combining data from all sources: email, calendar, and tasks. It should feel like a smart assistant has already read everything and written you a briefing before you woke up.

This story implements streaming — the briefing streams word-by-word into the widget as it generates, which creates a compelling live effect for the LinkedIn demo.

## Technical Instructions

### Step 1: Create lib/ai/prompts/briefing.ts

```typescript
import { ai } from '@/lib/ai';
import type { EmailThread, CalendarEvent, ObsidianTask } from '@/types';

const BRIEFING_SYSTEM_PROMPT = `You are ARIA, Doug's AI personal assistant. Generate a concise, professional morning briefing for Doug.

Doug is a full-time IT Support Specialist III at Zillow who is also an undergraduate STEM student. He values efficiency and clarity.

The briefing should:
- Be 3-4 short paragraphs
- Start with a greeting and today's date
- Cover: key emails needing attention, today's schedule highlights, top tasks to focus on
- Be written in second person ("You have...", "Your first meeting...")
- End with one motivating sentence
- Tone: professional but warm, like a trusted assistant
- Length: 150-200 words maximum`;

export async function generateMorningBriefing(data: {
  urgentEmails: EmailThread[];
  todayEvents: CalendarEvent[];
  topTasks: ObsidianTask[];
  agendaNarrative?: string;
  focusSuggestion?: string;
}): Promise<AsyncGenerator<string>> {
  const today = new Date().toLocaleDateString('en-US', {
    weekday: 'long', year: 'numeric', month: 'long', day: 'numeric'
  });

  const prompt = `Generate a morning briefing for ${today}.

URGENT EMAILS (${data.urgentEmails.length}):
${data.urgentEmails.slice(0, 3).map(e => `- From ${e.from}: ${e.summary ?? e.snippet}`).join('\n') || 'None'}

TODAY'S SCHEDULE (${data.todayEvents.length} events):
${data.todayEvents.slice(0, 5).map(e => `- ${new Date(e.start).toLocaleTimeString('en-US', { hour: 'numeric', minute: '2-digit' })}: ${e.title}`).join('\n') || 'Clear schedule'}

TOP TASKS:
${data.topTasks.slice(0, 3).map((t, i) => `${i + 1}. ${t.text}${t.dueDate ? ` (due ${t.dueDate})` : ''}`).join('\n') || 'No urgent tasks'}

${data.agendaNarrative ? `Schedule context: ${data.agendaNarrative}` : ''}
${data.focusSuggestion ? `Task context: ${data.focusSuggestion}` : ''}`;

  return ai.stream({
    system: BRIEFING_SYSTEM_PROMPT,
    prompt,
  });
}
```

### Step 2: Create the streaming briefing API route

Create `app/api/ai/briefing/route.ts`:

```typescript
import { auth } from '@/lib/auth';
import { fetchInboxThreads } from '@/lib/gmail';
import { fetchWeekEvents } from '@/lib/gcal';
import { readAllTasks } from '@/lib/obsidian';
import { generateMorningBriefing } from '@/lib/ai/prompts/briefing';
import { config } from '@/lib/config';

export async function GET() {
  const session = await auth();
  if (!session?.accessToken) {
    return new Response('Unauthorized', { status: 401 });
  }

  // Fetch all data sources in parallel
  const [rawThreads, rawEvents, rawTasks] = await Promise.allSettled([
    config.demoMode ? Promise.resolve(getMockEmails()) : fetchInboxThreads(session.accessToken, 5),
    config.demoMode ? Promise.resolve(getMockEvents()) : fetchWeekEvents(session.accessToken),
    config.demoMode ? Promise.resolve(getMockTasks()) : Promise.resolve(readAllTasks()),
  ]);

  const threads = rawThreads.status === 'fulfilled' ? rawThreads.value : [];
  const allEvents = rawEvents.status === 'fulfilled' ? rawEvents.value : [];
  const tasks = rawTasks.status === 'fulfilled' ? rawTasks.value : [];

  const today = new Date().toISOString().split('T')[0];
  const todayEvents = allEvents.filter(e => e.start.startsWith(today));
  const urgentEmails = threads.filter(t => t.urgency === 'high' || t.unread);

  // Return a streaming response (SSE / ReadableStream)
  const stream = await generateMorningBriefing({
    urgentEmails,
    todayEvents,
    topTasks: tasks.slice(0, 3),
  });

  const readableStream = new ReadableStream({
    async start(controller) {
      try {
        for await (const chunk of stream) {
          controller.enqueue(new TextEncoder().encode(chunk));
        }
      } finally {
        controller.close();
      }
    },
  });

  return new Response(readableStream, {
    headers: {
      'Content-Type': 'text/plain; charset=utf-8',
      'Cache-Control': 'no-cache',
      'Transfer-Encoding': 'chunked',
    },
  });
}

function getMockEmails() { return []; }
function getMockEvents() { return []; }
function getMockTasks() { return []; }
```

### Step 3: Update MorningBriefingWidget to stream

Update `components/widgets/MorningBriefingWidget.tsx`:

```tsx
'use client';

import { useEffect, useState } from 'react';
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card';
import { Sun } from 'lucide-react';

export function MorningBriefingWidget() {
  const [briefing, setBriefing] = useState('');
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState('');

  useEffect(() => {
    let cancelled = false;
    
    async function streamBriefing() {
      try {
        const response = await fetch('/api/ai/briefing');
        if (!response.ok) throw new Error('Failed to generate briefing');
        
        const reader = response.body?.getReader();
        if (!reader) throw new Error('No reader');
        
        const decoder = new TextDecoder();
        setLoading(false);
        
        while (true) {
          const { done, value } = await reader.read();
          if (done || cancelled) break;
          setBriefing(prev => prev + decoder.decode(value));
        }
      } catch (err) {
        if (!cancelled) setError('Unable to generate briefing right now.');
        setLoading(false);
      }
    }
    
    streamBriefing();
    return () => { cancelled = true; };
  }, []);

  return (
    <Card className="col-span-2 mb-4 min-h-[120px]">
      <CardHeader className="pb-2">
        <CardTitle className="text-sm font-medium flex items-center gap-2 text-muted-foreground">
          <Sun className="h-4 w-4 text-yellow-500" />
          Morning Briefing
        </CardTitle>
      </CardHeader>
      <CardContent>
        {loading && (
          <div className="flex gap-1 items-center text-muted-foreground text-sm">
            <span>Generating your briefing</span>
            <span className="animate-pulse">...</span>
          </div>
        )}
        {error && <p className="text-sm text-destructive">{error}</p>}
        {briefing && (
          <p className="text-sm leading-relaxed whitespace-pre-wrap">{briefing}</p>
        )}
      </CardContent>
    </Card>
  );
}
```

## Notes for Claude Code

- The streaming response uses the browser's native `ReadableStream` — no SSE library needed
- `Promise.allSettled` is used instead of `Promise.all` so one failing source doesn't block the briefing
- The briefing fetches raw data, not AI-enriched data (to avoid circular dependency) — it has its own AI call
- If the app is used after noon, the greeting and tone should adjust accordingly (the AI handles this via the date in the prompt)
