## Technical Instructions

### Step 1: Create lib/ai/prompts/calendar.ts

```typescript
import { ai } from '@/lib/ai';
import type { CalendarEvent } from '@/types';

const CALENDAR_SYSTEM_PROMPT = `You are ARIA's calendar intelligence processor. Analyze a list of calendar events and return structured JSON.

Always return valid JSON with this structure:
{
  "agendaNarrative": "2-3 sentence human-friendly summary of today's schedule and what to be aware of",
  "conflicts": [
    {
      "event1Id": "event-id-1",
      "event2Id": "event-id-2", 
      "description": "These two events overlap by 30 minutes"
    }
  ],
  "enrichedEvents": [
    {
      "id": "event-id",
      "agendaSummary": "Brief note or prep suggestion for this event",
      "hasConflict": false
    }
  ]
}

Conflict detection: Two events conflict if their time ranges overlap. Mark both events as hasConflict: true.
Agenda narrative: Be concise and helpful. Mention the total number of events, any conflicts, and what the busiest period is.
Event summaries: If an event title is vague (e.g., "Meeting"), suggest what to prepare. If it's clear, just confirm the time.`;

export interface CalendarAIResult {
  agendaNarrative: string;
  conflicts: Array<{ event1Id: string; event2Id: string; description: string }>;
  enrichedEvents: Array<{ id: string; agendaSummary: string; hasConflict: boolean }>;
}

export async function analyzeCalendarEvents(
  events: CalendarEvent[]
): Promise<CalendarAIResult> {
  if (events.length === 0) {
    return {
      agendaNarrative: 'Your schedule is clear. A great day to focus on deep work.',
      conflicts: [],
      enrichedEvents: [],
    };
  }

  // Only analyze today's events for the narrative; include all for conflicts
  const today = new Date().toISOString().split('T')[0];
  const todayEvents = events.filter(e => e.start.startsWith(today));

  const prompt = `Analyze these calendar events:

${events.map(e => `ID: ${e.id}
Title: ${e.title}
Start: ${e.start}
End: ${e.end}
Source: ${e.source}
`).join('\n')}

Today is ${today}. Events today: ${todayEvents.length}. Total events this week: ${events.length}.`;

  const result = await ai.complete({
    system: CALENDAR_SYSTEM_PROMPT,
    prompt,
    maxTokens: 768,
  });

  try {
    const json = result.text.replace(/```json\n?|\n?```/g, '').trim();
    return JSON.parse(json);
  } catch {
    return {
      agendaNarrative: `You have ${todayEvents.length} event(s) today and ${events.length} this week.`,
      conflicts: [],
      enrichedEvents: events.map(e => ({ id: e.id, agendaSummary: '', hasConflict: false })),
    };
  }
}
```

### Step 2: Update the calendar events route

In `app/api/calendar/events/route.ts`, after fetching events:

```typescript
import { analyzeCalendarEvents } from '@/lib/ai/prompts/calendar';

// After fetching allEvents:
const aiAnalysis = await analyzeCalendarEvents(allEvents).catch(() => null);

// Merge AI data into events
const enrichedEvents = allEvents.map(event => {
  const aiEvent = aiAnalysis?.enrichedEvents.find(e => e.id === event.id);
  return {
    ...event,
    agendaSummary: aiEvent?.agendaSummary,
    hasConflict: aiEvent?.hasConflict ?? false,
  };
});

return NextResponse.json({
  events: enrichedEvents,
  agendaNarrative: aiAnalysis?.agendaNarrative ?? null,
  conflicts: aiAnalysis?.conflicts ?? [],
});
```

### Step 3: Update CalendarWidget

Update `components/widgets/CalendarWidget.tsx` to:

1. **Agenda narrative:** Display `agendaNarrative` as a highlighted summary text at the top of the widget
2. **Conflict warnings:** If `conflicts.length > 0`, show a yellow warning banner: "⚠️ Schedule conflict detected"
3. **Conflict-marked events:** Show a red indicator next to events with `hasConflict: true`
4. **Event prep notes:** Show `agendaSummary` as subtitle text under each event title

## Notes for Claude Code

- The AI is given all events but focuses the narrative on today
- Conflict detection can also be done client-side, but doing it in the AI prompt means the model can describe conflicts in natural language
- Don't block event rendering on AI — load events first, then overlay AI data when it arrives
- The `agendaNarrative` and `conflicts` should be part of the API response, not stored separately
