# Story 4.3 — Demo-Safe Mock Data Mode

**Phase:** 4 — Polish & Demo Prep
**Depends on:** All integrations working
**Estimated time:** 1–2 hours

---

## Context

When recording the LinkedIn demo video, Doug doesn't want real emails, calendar events, or personal task names visible on screen. The demo mode replaces all real data with realistic-looking mock data so the video can be published publicly without privacy concerns.

Activating demo mode: set `DEMO_MODE=true` in `.env.local` and restart the server.

---

## What to Build

- A comprehensive mock data layer in `lib/mock-data.ts` with realistic-looking data
- Ensure all API routes check `config.demoMode` and return mock data when true
- A visible "Demo Mode" banner on the dashboard when active

---

### Step 2: Update all API routes to use mock data

Ensure the following routes return mock data when `config.demoMode === true`:
- `GET /api/email/threads` → `MOCK_EMAIL_THREADS`
- `GET /api/calendar/events` → `MOCK_CALENDAR_EVENTS`
- `GET /api/tasks` → `MOCK_TASKS`
- `GET /api/notes` → `MOCK_NOTES`
- `GET /api/ai/briefing` → stream `MOCK_MORNING_BRIEFING` character by character (to simulate streaming)

For the briefing stream in demo mode:
```typescript
if (config.demoMode) {
  const text = MOCK_MORNING_BRIEFING;
  const stream = new ReadableStream({
    async start(controller) {
      for (const char of text) {
        controller.enqueue(new TextEncoder().encode(char));
        await new Promise(r => setTimeout(r, 15)); // simulate streaming
      }
      controller.close();
    }
  });
  return new Response(stream, { headers: { 'Content-Type': 'text/plain' } });
}
```

### Step 3: Add Demo Mode banner

In `components/DashboardShell.tsx`, add a yellow banner below the TopBar when demo mode is active:

```tsx
{config.demoMode && (
  <div className="bg-yellow-500/20 border-b border-yellow-500/30 px-4 py-1 text-xs text-yellow-500 text-center">
    Demo Mode — showing sample data
  </div>
)}
```

Note: `config.demoMode` is server-side only. Pass it as a prop from the server component.
