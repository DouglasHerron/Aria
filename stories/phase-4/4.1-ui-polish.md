# Story 4.1 — UI Polish: Animations, Transitions & Responsive Layout

**Phase:** 4 — Polish & Demo Prep
**Depends on:** All Phase 1–3 stories complete and working
**Estimated time:** 2–3 hours

---

## Context

ARIA's UI works — now it needs to feel polished. This story adds subtle animations, loading transitions, and ensures the layout looks great at different screen sizes. The goal is a dashboard that looks like a real, professionally-built product — not a hackathon prototype.

Target aesthetic: Linear, Raycast, or Vercel dashboard — minimal, purposeful animations, no gimmicks.

---

## What to Polish

### 1. Widget entrance animations

When the dashboard loads, widgets should fade and slide up into position (staggered, one after another). Use Tailwind's `animate-in` utilities or install `framer-motion` for more control.

```bash
npm install framer-motion
```

Each widget should animate in with:
- Fade from opacity 0 → 1
- Slide from translateY(8px) → 0
- Staggered delay: Morning Briefing first (0ms), then Email and Calendar (150ms), then Tasks and Notes (300ms)

```tsx
// Example using framer-motion
<motion.div
  initial={{ opacity: 0, y: 8 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.3, delay: 0.15 }}
>
  <EmailWidget />
</motion.div>
```

### 2. Loading skeletons (refine existing)

Ensure every widget has a polished loading skeleton that matches the actual content shape:
- Email: 3 rows with avatar placeholder + text lines
- Calendar: timeline bars of varying width
- Tasks: 3 checkbox rows
- Notes: 2 title rows
- Morning Briefing: 3 paragraph lines

Use the shadcn/ui `Skeleton` component (add it if not already installed: `npx shadcn@latest add skeleton`).

### 3. Widget hover states

Add subtle hover effects to interactive elements:
- Email thread rows: `hover:bg-accent/50` with a smooth 150ms transition
- Task items: checkbox scales slightly on hover
- Calendar events: subtle left border color change
- All clickable rows should have `cursor-pointer`

### 4. Success/error toasts

Install and wire up the shadcn/ui toast component for all user actions:

```bash
npx shadcn@latest add toast sonner
```

Replace any inline success/error text with toast notifications:
- "Reply sent ✓" (green)
- "Task created ✓" (green)
- "Note saved ✓" (green)
- "Failed to send" (red)

Add `<Toaster />` to `app/layout.tsx`.

### 5. Responsive layout

The 2x2 widget grid should collapse to a single column on narrower viewports:

```tsx
<div className="grid grid-cols-1 lg:grid-cols-2 gap-4">
  {/* widgets */}
</div>
```

### 6. Sidebar active state

The sidebar should highlight the active section. Add state tracking so that when the Email Widget is scrolled into view, the Email icon in the sidebar highlights.

### 7. Dark/light mode toggle

Add a moon/sun toggle to the TopBar that toggles the `dark` class on `<html>`. Persist the preference in `localStorage`.

```tsx
// Toggle in TopBar
const toggleTheme = () => {
  const html = document.documentElement;
  html.classList.toggle('dark');
  localStorage.setItem('theme', html.classList.contains('dark') ? 'dark' : 'light');
};
```

## Notes for Claude Code

- Framer Motion's `AnimatePresence` is useful for mount/unmount transitions (e.g., email expand/collapse)
- Keep animations under 350ms — anything slower feels laggy
- The dark/light toggle should read from localStorage on app init (add to layout.tsx)
- Test at 1280px and 768px widths to verify the responsive breakpoint
- Don't add animations that loop or pulse constantly — only entrance and interaction feedback
